# External Researcher (Academic Literature Augmenter)

## Purpose
The `external-researcher` skill is the conditional Stage 3 in the 5-stage journal generation pipeline (Plan -> Analyze -> Augment -> Draft -> Review). It is invoked specifically for the **Related Work** and **Reference** sections. Unlike the standard `paper-analyzer` which only looks inward at the project codebase, this skill looks *outward* to the broader academic community.

## Input
- **YAML Plan:** The `docs/paper/Yantra/drafts/plan/plan_[section_name].yaml` containing the section's academic goals.
- **Analysis Document:** The `docs/paper/Yantra/drafts/details/[N]. [Section Name]/analysis_[section_name].md` generated by the `paper-analyzer`, containing the raw facts extracted from the internal documentation.

## Output
An augmented analysis document saved to `docs/paper/Yantra/drafts/details/[N]. [Section Name]/augmented_analysis_[section_name].md`.

## Responsibilities
1. **Academic Contextualization:** Read the internal facts from the `analysis` document (e.g., "The system uses Prefect for orchestration and DVC for data versioning").
2. **External Knowledge Retrieval:** Use your internal knowledge base (and web search if applicable via tools) to find the canonical academic papers, textbooks, or standard industry articles that define these concepts. **CRITICAL:** Any external reference, whether recalled from your LLM training or retrieved via web links, *must* be rooted in and explicitly reference **Q1 Scopus indexed journals** or universally accepted canonical texts.
   - **ESWA Citation Recency Constraint:** Ensure that 60-70% of the gathered citations are highly recent (published between 2022-2026).
   - **ESWA Relevance Constraint:** Actively search for and include at least 2-4 highly relevant papers published specifically in the *Expert Systems with Applications* journal.
3. **Comparative Baseline Generation:** For the "Related Work" section, identify 3-5 standard industry baselines or alternative academic approaches that tackle the same problem, setting up a formal comparison.
   - **ESWA Comparison Table Constraint:** Output the structural data required for a summary table comparing these baselines across specific dimensions: [Study, Approach, Dataset, Limitation].
4. **Augmentation:** Append this external intelligence to the original analysis, explicitly formatting citation targets so the `paper-drafter` knows exactly what to cite.

## Operating Principles
- **No Drafting:** Do not write the final Scopus-indexed prose. Only output the synthesized bullet points, facts, and formal citation targets.
- **Truthfulness & Rigor:** Be extremely precise with author names, publication years, and methodologies when augmenting the analysis. Do not invent fake academic papers. Standardize all gathered citations to demonstrate a Q1 Scopus-level of academic rigor.
