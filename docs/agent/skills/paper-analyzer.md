# Paper Analyzer (Cross-Library Data Synthesizer)

## Purpose
The `paper-analyzer` skill is the second stage in the 4-stage journal generation pipeline (Plan -> Analyze -> Draft -> Review). It executes the analysis instructions defined in a specific section's YAML plan (`plan_[section_name].yaml`). It synthesizes the technical data required for drafting.

## Input
- **YAML Plan:** The `docs/paper/Yantra/drafts/plan/plan_[section_name].yaml` generated by the `paper-planner` (and potentially modified by the user).

## Output
A detailed, synthesized raw data document saved to `docs/paper/Yantra/drafts/details/[N]. [Section Name]/analysis_[section_name].md`.

## Responsibilities
1. **Targeted Reading:** Strictly read only the files specified in the `data_sources` section of the provided YAML plan.
2. **Data Extraction:** Extract the specific information requested (e.g., math logic, performance metrics, system interaction patterns) from the Amsha, Yantra, Bodha, and cross-module documentation layers.
3. **Synthesis:** Combine the extracted data points into a coherent set of facts, models, dependencies, and evidence. 
4. **Adhere to Instructions:** Follow the explicit rules defined in `instructions.analyzer` within the YAML plan.

## Operating Principles
- **Strict Compliance:** Do not pull data from sources not explicitly listed in the YAML plan. 
- **No Fluff:** The output should be dense, factual, and strictly technical. It is a data feed for the drafter, not a polished prose document. 
- **Preserve Formalisms:** Ensure that any extracted LaTeX math equations or Mermaid diagrams are preserved verbatim for the drafter to format.
- **No Hallucination:** If a data point requested by the YAML plan cannot be found in the specified source paths, note the missing data explicitly rather than inventing it.
